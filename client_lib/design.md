# Client Lib Design

## Назначение

`client_lib` — это интерфейсный слой между TCP-клиентом и пользовательскими приложениями (`client console`, `client gui`).

Его задача:
- подключаться к серверу,
- сообщать о состоянии соединения,
- запрашивать snapshot,
- передавать обновления top-of-book,
- уведомлять об ошибках.

Этот слой скрывает сетевые детали и даёт UI удобный API.

---

## Основные сущности

### `TopLevel`
Структура верхнего уровня стакана.  
Используется для передачи актуального состояния лучших цен.

### `Snapshot`
Снимок состояния рынка на текущий момент.  
Обычно нужен сразу после подключения или по отдельному запросу.

### `ConnectionState`
Описывает текущее состояние клиента:
- `Stopped` — клиент остановлен;
- `Connecting` — идёт подключение;
- `Connected` — соединение установлено;
- `Reconnecting` — выполняется переподключение;
- `Disconnected` — соединение отсутствует.

### `ClientError`
Перечисление возможных ошибок клиентской библиотеки:
- ошибки сокета,
- ошибки подключения,
- ошибки чтения и записи,
- ошибки протокола,
- ошибки запроса snapshot.

### `ClientConfig`
Конфигурация подключения:
- `host` — адрес сервера;
- `port` — порт сервера;
- `auto_reconnect` — нужно ли автоматически переподключаться;
- `reconnect_delay` — задержка между попытками переподключения.

### `ClientCallbacks`
Набор callback-функций для уведомления пользователя библиотеки о событиях:
- подключение,
- отключение,
- смена состояния,
- получение snapshot,
- получение top-of-book,
- ошибка.

## Интерфейс `IOrderBookClient`

`IOrderBookClient` задаёт единый контракт клиентской библиотеки.

### Основные методы

- `SetCallbacks(...)`  
  Устанавливает обработчики событий.

- `Connect(...)`  
  Настраивает подключение к серверу.

- `Disconnect()`  
  Разрывает соединение.

- `Start()`  
  Запускает работу клиента.

- `Stop()`  
  Останавливает работу клиента.

- `RequestSnapshot()`  
  Запрашивает snapshot у сервера.

- `IsConnected()`  
  Показывает, установлено ли соединение.

- `State()`  
  Возвращает текущее состояние клиента.

---

## Сценарий работы

1. Пользователь создаёт реализацию `IOrderBookClient`.
2. Передаёт callback-функции через `SetCallbacks`.
3. Вызывает `Connect(config)`.
4. Запускает клиента через `Start()`.
5. После подключения получает `on_connected`.
6. При необходимости запрашивает `RequestSnapshot()`.
7. Получает `on_snapshot` и `on_top_of_book`.
8. При ошибках получает `on_error`.
9. При разрыве соединения получает `on_disconnected`.
10. Если включён `auto_reconnect`, клиент пытается восстановить соединение.

## Идея архитектуры

Интерфейс отделяет:
- транспортный уровень (TCP),
- прикладную клиентскую библиотеку,
- пользовательский интерфейс.

Это позволяет:
- писать `console` и `gui` независимо от сетевой реализации;
- позже заменить реализацию клиента без изменения UI;
- удобно тестировать совместимость по единому API.
